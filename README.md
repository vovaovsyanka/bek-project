# Prompt Injection Defense Service

Сервис для обучения защите от промпт-инъекций LLM. Запросы проходят через каскад классификаторов с разной архитектурой, а затем обрабатываются LLM в стиле призраков.

## Архитектура пайплайна

```
Пользовательский запрос
    |
    v
+------------------+
| Общий BERT       | --> БЕЗОПАСНО --> LLM --> Ответ пользователю
| (опасно/безопасно)|
+------------------+
    |
    ОПАСНО
    |
    v
+-------------------------+
| Категориальный BERT     | --> НЕ ЦЕЛЕВАЯ --> "Категория не соответствует"
| (целевая категория 3/4/5)|
+-------------------------+
    |
    ЦЕЛЕВАЯ
    |
    v
+-------------------------+
| Классификатор уровня    | --> ОПАСНО --> "Модель поняла, запрос нужно усилить"
| 1-TF-IDF / 2-LSTM /     |
| 3-BERT                  |
+-------------------------+
    |
    БЕЗОПАСНО
    |
    v
   LLM --> Ответ пользователю
```

## Структура проекта

```
prompt_injection_service/
  main.py                  # FastAPI бэкенд (endpoint /process)
  pipeline.py              # Логика пайплайна обработки
  llm_client.py            # Клиент для Ollama API
  base_classifier.py       # Базовые классификаторы (TF-IDF, LSTM, BERT)
  category_classifier.py   # Категориальные BERT классификаторы
  static/
    index.html             # Фронтенд (поле ввода, выбор параметров, вывод)
  requirements.txt         # Зависимости Python
  README.md                # Этот файл
```

## Установка и запуск

### 1. Клонирование и подготовка окружения

```bash
# Перейдите в директорию проекта
cd prompt_injection_service

# Создайте виртуальное окружение (рекомендуется)
python -m venv venv

# Активируйте окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### 2. Настройка моделей классификаторов

#### 2.1 Общий BERT классификатор (`base_classifier.py`)

Модель должна быть размещена по пути:
```
classifiers/models/rubert/
```

Это дообученная модель `ai-forever/ru-en-RoSBERTa` (или аналог), классифицирующая запросы на `опасный`/`безопасный`.

#### 2.2 Категориальные классификаторы (`category_classifier.py`)

Для каждой категории (3, 4, 5) требуется отдельная обученная модель.

**Настройте пути в файле `category_classifier.py`**:

```python
CATEGORY_MODEL_PATHS = {
    3: "models/category_3",   # <-- Укажите путь к модели категории 3
    4: "models/category_4",   # <-- Укажите путь к модели категории 4
    5: "models/category_5",   # <-- Укажите путь к модели категории 5
}
```

Каждая модель — это дообученный BERT, определяющий принадлежность запроса к целевой категории (бинарная классификация: target vs other).

Если модель не найдена по указанному пути, классификатор перейдёт в **режим заглушки** (все запросы считаются целевыми) и выведет предупреждение в консоль.

#### 2.3 Классификаторы уровней (`base_classifier.py`)

| Уровень | Классификатор | Путь к модели |
|---------|---------------|---------------|
| 1 | TF-IDF + линейная модель | `classifiers/models/tfidf_classifier.pkl` |
| 2 | LSTM | `classifiers/models/security_lstm_pytorch.pth` |
| 3 | BERT | `classifiers/models/rubert/` (общий) |

Убедитесь, что все веса моделей размещены по соответствующим путям.

### 3. Настройка переменных окружения

Создайте файл `.env` в корне проекта или экспортируйте переменные:

```bash
# Ollama API
OLLAMA_API_KEY=your_api_key_here        # API ключ для Ollama
OLLAMA_MODEL=qwen3-vl:235b-cloud        # Название модели (по умолчанию)
OLLAMA_API_URL=https://ollama.com/api/generate  # URL API Ollama

# Опционально: настройки сервера
HOST=0.0.0.0                            # Хост для запуска
PORT=8000                               # Порт
```

### 4. Запуск сервиса

```bash
# Запуск через uvicorn (рекомендуется для разработки)
uvicorn main:app --reload --host 0.0.0.0 --port 8000

# Или напрямую
python main.py
```

Сервис будет доступен по адресу: http://localhost:8000

### 5. Доступ к фронтенду

Откройте в браузере:
```
http://localhost:8000/
```

## API

### POST `/process`

Основной endpoint для обработки запросов.

**Тело запроса (JSON):**
```json
{
    "query": "Текст вашего промпт-запроса",
    "category": 4,
    "difficulty": 2
}
```

- `query` — текст запроса (строка, обязательное)
- `category` — целевая категория: `3`, `4` или `5` (integer, обязательное)
- `difficulty` — уровень сложности: `1` (TF-IDF), `2` (LSTM), `3` (BERT)

**Ответ (JSON):**
```json
{
    "response": "Текстовый ответ от сервиса",
    "status": "ok"
}
```

### GET `/health`

Проверка работоспособности сервиса.

**Ответ:**
```json
{
    "status": "ok",
    "pipeline_ready": true
}
```

## Возможные ответы сервиса

| Ситуация | Ответ |
|----------|-------|
| Общий BERT: безопасно | Ответ от LLM в стиле призрака |
| Общий BERT: опасно → Категория: не целевая | "Категория вашего запроса не соответствует выбранной..." |
| Общий BERT: опасно → Категория: целевая → Уровень: опасно | "Модель (X) поняла, что запрос опасный и его нужно усилить..." |
| Общий BERT: опасно → Категория: целевая → Уровень: безопасно | Ответ от LLM в стиле призрака |

## Фронтенд

Веб-интерфейс доступен по корневому URL (`/`). Содержит:
- Поле ввода запроса
- Выпадающий список выбора категории (3, 4, 5)
- Выпадающий список выбора уровня сложности (1, 2, 3)
- Кнопку отправки
- Поле вывода ответа сервиса

## Используемые технологии

- **Backend**: FastAPI, Uvicorn
- **ML/DL**: PyTorch, Transformers (Hugging Face), scikit-learn
- **LLM**: Ollama API (Qwen3-VL)
- **Frontend**: HTML, CSS, Vanilla JavaScript

## Примечания

- Категории 3, 4, 5 соответствуют:
  - **3** — Прямой запрос секретного слова/данных
  - **4** — Манипуляция, социальная инженерия, обман
  - **5** — Косвенный запрос, обходные формулировки

- Уровни сложности:
  - **1** — TF-IDF + линейная модель (самый простой классификатор)
  - **2** — LSTM (рекуррентная нейросеть)
  - **3** — BERT (трансформер, самый сложный)
